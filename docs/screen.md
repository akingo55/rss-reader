# 画面設計書

RSSリーダーに必要な画面、表示項目、操作、状態を整理するための設計書です。
実装時はこのドキュメントを画面仕様の基準とし、DB設計書の `feeds`、`articles`、`profiles`、`user_feeds`、`article_reads` と対応させます。

## 0. 共通方針

### サービス名

- 画面上のサービス名は `rss-reader` に統一します。

### 認証

- 認証は GitHub OAuth のみを提供します。
- 認証基盤は Supabase Auth を利用します。
- 未ログインユーザーは、公開画面のみ閲覧できます。
- 購読、購読解除、フィード追加、既読登録、プロフィール管理などユーザーに紐づく操作はログイン必須です。
- ログインが必要な画面へ未ログイン状態でアクセスした場合は、ホーム画面へ誘導し、GitHubログイン導線を表示します。

### 共通レイアウト

- ログイン中は画面左側、またはモバイルではメニュー内にナビゲーションを表示します。
- ナビゲーションには以下を配置します。
  - ホーム
  - フィード一覧
  - 購読フィード管理
  - プロフィール管理
- ログイン中はユーザー情報とログアウトボタンを表示します。
- ログアウト後はホーム画面へリダイレクトします。

### 画面状態

各画面は、必要に応じて以下の状態を用意します。

- 読み込み中: データ取得中であることを表示します。
- 空状態: 表示対象がない場合に、次の行動につながる案内を表示します。
- エラー状態: データ取得や更新に失敗した場合に、再試行できる表示を行います。
- 権限なし: ログインが必要な操作では、ログインを促します。

### 日時表示

- 記事の公開日時は `published_at` を表示します。
- `published_at` が未設定の場合は「公開日時不明」と表示します。
- 日時は日本語ユーザー向けに、年月日と時刻が分かる形式で表示します。

### 外部リンク

- サイトURL、記事URLは外部リンクとして扱います。
- 記事URLを開く場合は新しいタブで表示します。
- 購読フィード詳細画面で記事URLを開いた場合、クリックイベントをもって既読登録します。
- 一度既読になった記事は、未読状態には戻せません。

### 記事一覧

- 記事一覧は1ページ30件表示とし、31件以上ある場合はページネーションを表示します。
- 記事一覧には並び順を選択するドロップダウンリストを表示します。
- 並び順の初期値は公開日時の新しい順です。
- 並び順の選択肢は、公開日時の新しい順、公開日時の古い順を提供します。
- `published_at` が未設定の記事は、同じ並び順の末尾に表示します。

## 1. 画面一覧

| 画面 | 想定ルート | ログイン | 主な目的 |
| --- | --- | --- | --- |
| ホーム画面 | `/` | 任意 | サービス説明、ログイン導線、ログイン中の概要表示 |
| フィード一覧画面 | `/feeds` | 任意 | 登録済みフィードの検索、購読、フィード追加 |
| フィード詳細画面 | `/feeds/[feedId]` | 任意 | フィードに紐づく記事一覧の閲覧、購読 |
| 購読フィード詳細画面 | `/myfeeds/[feedId]` | 必須 | 購読中フィードの記事閲覧、既読管理 |
| 購読フィード管理画面 | `/myfeeds` | 必須 | 購読中フィードの一覧確認、購読解除 |
| プロフィール管理画面 | `/profile` | 必須 | ユーザー情報の確認 |

## 2. ホーム画面

### 目的

未ログインユーザーにはサービスの概要とログイン導線を表示し、ログインユーザーには利用開始しやすい概要情報を表示します。

### 未ログイン時の表示項目

- サービス名 `rss-reader`
- RSSリーダーの概要説明
- GitHubでログインするボタン
- フィード一覧画面への導線

### ログイン時の表示項目

- サービス名 `rss-reader`
- ログイン中ユーザーの表示名、メールアドレス、またはユーザーID
- ユーザーが購読中のフィード一覧の一部
- フィード一覧画面への導線
- 購読フィード管理画面への導線
- ログアウトボタン

### 操作

- GitHubでログインするボタンを押すと、GitHub OAuth フローを開始します。
- ログアウトボタンを押すとログアウトし、ホーム画面に戻ります。
- 購読中フィードを選択すると、購読フィード詳細画面へ遷移します。

### 空状態

- ログイン中で購読フィードがない場合は、フィード一覧画面へ誘導します。

## 3. フィード一覧画面

### 目的

登録済みの全フィードを一覧表示し、ユーザーが検索、購読、未登録フィードの追加を行えるようにします。
フィード追加はログインユーザーのみ可能です。

### 表示項目

- フィード検索フォーム
- フィード追加ボタン
- フィード一覧
  - フィードタイトル
  - RSS URL
  - サイトURLへのリンク
  - 購読状態
  - 購読ボタン、または購読中表示

### 操作

- 検索フォームに入力すると、タイトル、RSS URL、サイトURLを対象にフィードを絞り込みます。
- 未ログインユーザーが購読ボタンを押した場合は、ログインを促します。
- ログインユーザーが購読ボタンを押した場合は、そのフィードを購読します。
- フィードタイトルを選択すると、フィード詳細画面へ遷移します。
- ログインユーザーがフィード追加ボタンを押すと、フィード追加ダイアログを表示します。
- 未ログインユーザーがフィード追加を行おうとした場合は、ログインを促します。

### フィード追加ダイアログ

#### 入力項目

- RSS URL

#### 操作

- 登録ボタンを押すと、入力されたRSS URLを登録します。
- キャンセルボタン、またはダイアログ外の操作で閉じます。

#### バリデーション

- RSS URLは必須です。
- URL形式ではない場合はエラーを表示します。
- すでに登録済みのRSS URLの場合は、新規登録せず既存フィードを案内します。
- RSS取得や解析に失敗した場合は、登録できなかった理由を表示します。

### 空状態

- 登録済みフィードがない場合は、フィード追加を促します。
- 検索結果が0件の場合は、検索条件の変更またはフィード追加を促します。

## 4. フィード詳細画面

### 目的

フィードに紐づく記事一覧を表示し、未購読ユーザーでもフィード内容を確認できるようにします。

### 表示項目

- フィードタイトル
- RSS URL
- サイトURLへのリンク
- 購読ボタン、または購読中表示
- 記事一覧
  - 記事タイトル
  - 記事URLへのリンク
  - 公開日時
  - 記事概要
- 記事一覧の並び順ドロップダウンリスト
- ページネーション

### 操作

- 未ログインユーザーが購読ボタンを押した場合は、ログインを促します。
- ログインユーザーが購読ボタンを押した場合は、そのフィードを購読します。
- 記事タイトル、または記事URLを押すと、記事URLを新しいタブで開きます。
- 並び順を変更すると、選択した条件で記事一覧を再表示します。
- ページネーションを操作すると、選択したページの記事一覧を表示します。

### 空状態

- フィードに記事がない場合は、記事がまだ取得されていないことを表示します。

### エラー状態

- 存在しないフィードIDの場合は、フィードが見つからないことを表示します。

## 5. 購読フィード詳細画面

### 目的

ログインユーザーが購読しているフィードの記事を読み、既読状態を管理できるようにします。

### 表示項目

- フィードタイトル
- サイトURLへのリンク
- 購読中表示
- 記事一覧
  - 記事タイトル
  - 記事URLへのリンク
  - 公開日時
  - 記事概要
  - 既読、未読の表示
- 記事一覧の並び順ドロップダウンリスト
- ページネーション

### 操作

- 記事タイトル、または記事URLを押すと、記事URLを新しいタブで開きます。
- 記事URLを開いたタイミングで、その記事を既読として `article_reads` に登録します。
- すでに既読登録済みの記事を開いた場合は、重複登録せず既読状態を維持します。
- 既読記事を未読に戻す操作は提供しません。
- 並び順を変更すると、選択した条件で記事一覧を再表示します。
- ページネーションを操作すると、選択したページの記事一覧を表示します。
- フィード一覧へ戻る導線を表示します。

### 表示ルール

- 未読記事は既読記事よりも目立つ表示にします。
- 既読記事はタイトルや背景を控えめにし、読了済みであることが分かるようにします。
- 既読日時を表示する場合は `read_at` を利用します。

### 空状態

- 記事がない場合は、記事がまだ取得されていないことを表示します。

### 権限なし

- ログインしていない場合はホーム画面へ誘導します。
- ログイン中でも購読していないフィードIDへアクセスした場合は、フィード詳細画面へ誘導します。

## 6. 購読フィード管理画面

### 目的

ログインユーザーが購読中のフィードを一覧で確認し、購読解除できるようにします。

### 表示項目

- 購読中フィード一覧
  - フィードタイトル
  - RSS URL
  - サイトURLへのリンク
  - 購読開始日時
  - 購読解除ボタン

### 操作

- フィードタイトルを選択すると、購読フィード詳細画面へ遷移します。
- 購読解除ボタンを押すと、確認ダイアログを表示します。
- 確認ダイアログで実行すると、そのフィードの購読を解除します。
- 購読解除後もフィード本体と記事は削除しません。削除するのは `user_feeds` のユーザーとフィードの関連のみです。

### 空状態

- 購読中フィードがない場合は、フィード一覧画面へ誘導します。

## 7. プロフィール管理画面

### 目的

ログインユーザーが自身のプロフィール情報を確認できるようにします。

### 表示項目

- 表示名
- メールアドレス
- アバター画像
- ユーザーID
- ログアウトボタン

### 操作

- ログアウトボタンを押すとログアウトし、ホーム画面へ戻ります。

### 補足

- GitHub認証ではメールアドレスを取得できない場合があります。その場合はメールアドレス未設定として表示します。
- 現段階ではプロフィール編集機能は提供せず、参照のみとします。

## 8. データ取得と更新

### 主な取得データ

- 全フィード一覧: `feeds`
- フィード詳細: `feeds` と紐づく `articles`
- ログインユーザー情報: `profiles`
- 購読中フィード一覧: `user_feeds` と紐づく `feeds`
- 既読状態: `article_reads`

### 主な更新操作

- フィード購読: `user_feeds` に `profile_id` と `feed_id` を登録します。
- フィード購読解除: `user_feeds` から `profile_id` と `feed_id` の関連を削除します。
- 記事既読登録: `article_reads` に `profile_id`、`article_id`、`read_at` を登録します。一度登録した既読状態は解除しません。
- フィード追加: ログインユーザーのみ実行できます。`feeds` にRSS URL、タイトル、サイトURLを登録し、取得した記事を `articles` に登録します。

## 9. レスポンシブ対応

- デスクトップでは、ナビゲーションを左側に固定表示します。
- モバイルでは、ナビゲーションをヘッダー内のメニューとして表示します。
- フィード一覧と記事一覧は、モバイルでもタイトル、状態、主要操作が読みやすいように縦並びにします。
- 外部リンク、購読ボタン、購読解除ボタンはタップしやすいサイズを確保します。

## 10. アクセシビリティ

- ボタン、リンク、入力欄には用途が分かるラベルを付けます。
- ダイアログ表示中は、キーボード操作で入力、登録、キャンセルができるようにします。
- エラーメッセージは入力欄の近くに表示します。
- 色だけで購読中、未読、既読を判別させず、テキストでも状態を表示します。
